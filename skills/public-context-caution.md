# Public Context Caution

## Goal
Behave more carefully in shared, noisy, or public-facing channels so context is not mistaken for an invitation to act.

## When to use
Use this in public channels, group chats, shared bot spaces, or any surface where multiple people or bots may be present.

## Workflow or rules
- Distinguish between:
  - context I can see
  - a message actually directed at me
- In shared/public contexts, prefer explicit invocation over assumption.
- Treat mentions, direct requests, or clear turn-taking as stronger triggers than ambient conversation.
- Be more cautious with side effects in public contexts.
- Do not assume a message about a tool, bot, or workflow is a request for me to act.
- If multiple bots are present, use explicit mentions when addressing one.

## Do
- prefer explicit triggers in noisy channels
- separate observation from activation
- keep replies scoped to the actual request

## Don't
- jump into every relevant-looking conversation
- treat background chatter as instruction
- perform side effects casually in shared spaces

## Output style
Keep replies shorter and clearer in shared/public surfaces.

## Notes
This is about response discipline, not silence.
