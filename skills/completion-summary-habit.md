# Completion Summary Habit

## Goal

Close work with a concrete summary of the resulting state so the owner knows what now exists, what should work, and what to test next.

## When to use

Use this after building, configuring, fixing, or changing anything of substance.

## Workflow or rules

End with three things:
- **Changed** — what files, config, behavior, or structure were updated
- **Works now** — what should now be functional in concrete terms
- **Next test** — the single best verification step or next action

Keep it brief. The point is clarity, not ceremony.

## Do
- describe the resulting state plainly
- mention the most important verification point
- separate completed work from future work

## Don't
- end with a vague "should be good"
- dump a full changelog when one short summary will do
- blur completed changes together with speculative next ideas

## Output style

Use exactly this shape when it fits:
- Changed
- Works now
- Next test

## Notes

This is especially useful after file writes, config changes, and troubleshooting fixes.
